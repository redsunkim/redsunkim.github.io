제조업 고객을 대상으로 **Datadog 기반 클라우드 ITO(IT 운영) 제안서**를 만들 때, **ChatGPT(내용/논리) + Gemini(슬라이드 자동 생성/편집)**를 역할 분담하면 “전문 제안서 초안”을 1시간 내로 만들 수 있습니다.

이 글은 그 핵심 워크플로우와 바로 복붙 가능한 프롬프트를 정리한 실전 가이드입니다.

---

## 1. 배경: 왜 제조업 클라우드 운영은 어려운가?

* **복잡성 급증**: IT/OT 융합 + 공장/본사 분산 + 하이브리드(AWS+Onprem) 환경
* **난이도 상승**: K8s 기반 신규 서비스(EKS/온프레 K8s) 확대로 릴리즈 및 장애 대응이 어려움
* **비용 이슈**: 클라우드 비용 증가율은 높은데, K8s 워크로드 단위 비용 귀속이 어려움 (FinOps 부재)
* **규제 대응**: ISMS 감사 및 규정 대응을 위한 보안·운영 증적 자동화 필요

## 2. 목표: Datadog 기반 “단일 관제 + 자동화 + 비용/보안 내재화”

* **Unified View**: Datadog을 단일 Observability/ITO 허브로 두고 AWS와 Onprem을 수평 통합
* **SRE 방식 운영**: K8s 신규 서비스 안정 운영 (SLI/SLO, 배포 회귀 탐지, 자동 복구)
* **FinOps 내재화**: 비용-성능 데이터를 연결해 증가율 원인 추적 및 최적화
* **ISMS 대응**: 보안 이벤트, 구성 변경, 민감 정보를 관제 데이터와 연결하여 증적 자동화

---

## 3. 전체 워크플로우 (Summary)

1.  **ChatGPT**: 제안서 논리 구조 및 슬라이드별 원고(JSON) 생성
2.  **Gemini Canvas**: 원고를 바탕으로 Google Slides 덱 자동 생성 및 Export
3.  **Google Slides (w/ Gemini)**: 내용 확장, 톤앤매너 통일, 핵심 도식 생성
4.  **Human**: 최소한의 디자인 룰 마감

---

## Step 1. ChatGPT에서 “슬라이드 설계도” 만들기

먼저 ChatGPT에게 제안서의 뼈대를 요청합니다.

### 1-1. 목차 및 스토리라인 프롬프트

```text
너는 B2B 컨설팅 제안서 디렉터야.
주제: Datadog 기반 클라우드 ITO 제안
고객: 제조업 일반 고객
전제:
- 대상: K8s 기반 신규 서비스(EKS/온프레 K8s)
- 환경: AWS + On-premise 하이브리드
- Pain: 클라우드 비용 증가율
- 규정: ISMS

1) 15~18장 제안서 목차를 문제-해결-가치-실행-리스크-비용-다음단계 흐름으로 만들어줘.
2) 각 슬라이드 결론형 헤드라인 1줄 + 핵심 불릿 3~5개 써줘.
3) 임원용 톤, 과장 없이 수치/근거 중심.
````

### 1-2. 슬라이드 원고를 JSON으로 구조화

목차가 나오면, 이를 Gemini Canvas나 스크립트에서 활용하기 좋게 JSON으로 변환합니다.

Plaintext

```
위 목차를 기반으로 슬라이드별 원고를 JSON으로 구체화해줘.

형식:
{
 "slide_no": n,
 "title": "...",
 "key_message": "...",
 "bullets": ["...","..."],
 "visual_hint": "추천 도표/그림/레이아웃",
 "speaker_notes": "발표자 노트 80~120자"
}
수치/가정은 합리적 범위에서 추정하고 (추정) 표시해줘.
```

## Step 2. Gemini Canvas로 “제안서 덱 자동 생성”

Google Workspace의 Gemini Canvas(또는 Gemini 사이드바)를 활용해 가장 빠르게 초안 덱을 만듭니다.

### 덱 생성 프롬프트

Plaintext

```
아래 내용을 기반으로 제조업 일반 고객 대상 ‘Datadog 기반 클라우드 ITO 제안서’ Google Slides 덱(15~18장)을 만들어줘.

전제:
- 대상: K8s 신규 서비스(EKS/온프레 K8s)
- 환경: AWS + On-premise 하이브리드
- Pain: 클라우드 비용 증가율
- 규정: ISMS

스타일:
- 임원용 B2B 컨설팅 톤, 미니멀, 화이트 배경
- 포인트 컬러 1개(블루 계열)
- 각 슬라이드는 결론형 헤드라인 1줄 + 불릿 3~5개
- 도표/아키텍처/대시보드는 placeholder로 넣기
- 마지막에 Google Slides로 Export 가능하게

슬라이드 원고(JSON):
(여기에 ChatGPT가 만들어준 JSON 내용을 그대로 붙여넣기)
```

> **Tip**: 생성 완료 후 `Export to Google Slides` 버튼을 눌러 슬라이드로 변환합니다.

## Step 3. Google Slides에서 Gemini로 마감 자동화

생성된 슬라이드에서 우측 사이드바의 Gemini를 열고 디테일을 다듬습니다.

### 3-1. 흐름 점검 & 부족한 장 추가

Plaintext

```
이 덱 흐름을 검토하고 제조업/K8s/AWS-Onprem/FinOps/ISMS 관점에서 빠진 내용이 있으면 2장 이내로 추가해줘.
추가 장도 결론형 헤드라인과 불릿으로 작성해줘.
```

### 3-2. 슬라이드 톤 통일 (Rewrite)

Plaintext

```
임원용 제안서 톤으로 더 간결하고 단호하게 다듬어줘.
불릿은 4개 이하, 동사형/결론형으로 수정해줘.
```

### 3-3. Executive Summary 생성

Plaintext

```
전체 덱을 2장짜리 Executive Summary로 요약해줘.
1장은 문제/해결/기대효과, 2장은 PoC/로드맵/다음단계로 구성해줘.
```

## Step 4. “전문 제안서 느낌”을 만드는 3대 핵심 그림

텍스트만 있는 제안서는 설득력이 떨어집니다. Gemini에게 이미지 생성을 요청하거나, 도식화 아이디어를 받아 그립니다.

### 4-1. 하이브리드 아키텍처 (AWS+Onprem → Datadog)

Plaintext

```
AWS(EKS/EC2/RDS) + On-premise(공장/본사 시스템) + OT 연계가 Datadog 단일 Observability 허브로 연결되는 하이브리드 아키텍처 다이어그램을 만들어줘. 미니멀, 아이콘 중심.
```

### 4-2. K8s 운영 폐루프 (SRE)

Plaintext

```
K8s 운영 폐루프(관측 → SLO기반 탐지 → 자동조치 → RCA → 개선)를 한 장짜리 순환 다이어그램으로 그려줘.
```

### 4-3. 비용-성능 상관 흐름 (FinOps)

Plaintext

```
클라우드 비용 증가율 원인을 ‘워크로드 비용 귀속 → 성능/트레이스 상관 → 낭비 후보 제거’로 보여주는 2축 상관 그래프/흐름도를 만들어줘.
```

---

## Step 5. 최소 디자인 룰 (이것만 지켜도 컨설팅 톤)

사람의 손길은 마지막 디자인 마감에만 사용합니다.

1. **헤드라인**: 결론형 한 문장 ("~합니다", "~가능합니다", "~필요합니다")
    
2. **메시지**: 한 장에 메시지 1개 원칙
    
3. **불릿**: 3~5개, 각 1줄 길이 유지
    
4. **컬러**: 폰트 1개 + 포인트 컬러 1개 (블루 등)
    
5. **여백**: 텍스트 박스 폭을 줄여 여백을 충분히 확보
    

---

## 6. 결론: 1시간 제안서 루틴

이 방식의 핵심은 **내용-슬라이드-리라이트-도식 생성**이 하나의 흐름으로 연결되어 제안서 초안 생산 속도를 극대화한다는 점입니다.

|**단계**|**소요 시간**|**작업 내용**|
|---|---|---|
|**1. ChatGPT**|10~15분|목차/헤드라인/불릿 생성 및 JSON 정리|
|**2. Gemini Canvas**|5분|덱 생성 & Slides Export|
|**3. Slides (Gemini)**|15~20분|내용 확장, 톤 통일, 그림 생성|
|**4. Human**|10분|숫자(추정) 표기, 용어 통일, 정렬 마감|
|**Total**|**약 50분**|**제안서 초안 완성**|

**Next Step**: ChatGPT JSON을 Apps Script와 연동하면 원고에서 슬라이드 생성까지 완전 자동화("원클릭 생성")도 가능합니다.
